<h1><a id="user-content-running-zigbee2mqtt" class="anchor" aria-hidden="true" href="#running-zigbee2mqtt"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Running Zigbee2MQTT</h1>
<p>These instructions explain how to run Zigbee2MQTT on bare-metal Linux.</p>
<p>You can also run Zigbee2MQTT in a <a href="/JsBergbau/zigbee2mqtt.io/blob/master/docs/information/docker.md">Docker container</a>, as the <a href="https://github.com/zigbee2mqtt/hassio-zigbee2mqtt">Home Assistant Zigbee2MQTT add-on</a>, in a <a href="/JsBergbau/zigbee2mqtt.io/blob/master/docs/information/virtual_environment.md">Python Virtual Enviroment</a> or even on <a href="/JsBergbau/zigbee2mqtt.io/blob/master/docs/information/windows.md">Windows</a>.</p>
<p>For the sake of simplicity this guide assumes running on a Raspberry Pi 3 with Raspbian Stretch Lite, but will work on any Linux machine.</p>
<p>Before starting make sure you have an MQTT broker installed on your system.
There are many tutorials available on how to do this, <a href="https://randomnerdtutorials.com/how-to-install-mosquitto-broker-on-raspberry-pi/" rel="nofollow">example</a>.
Mosquitto is the recommended MQTT broker but others should also work fine.</p>
<h2><a id="user-content-1-determine-location-of-the-adapter-and-checking-user-permissions" class="anchor" aria-hidden="true" href="#1-determine-location-of-the-adapter-and-checking-user-permissions"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>1. Determine location of the adapter and checking user permissions</h2>
<p>We first need to determine the location of the adapter. Connect the adapter to your Raspberry Pi. Most of the times the location is <code>/dev/ttyACM0</code>. This can be verified by:</p>
<div class="highlight highlight-source-shell"><pre>pi@raspberry:<span class="pl-k">~</span> $ ls -l /dev/ttyACM0
crw-rw---- 1 root dialout 166, 0 May 16 19:15 /dev/ttyACM0  <span class="pl-c"><span class="pl-c">#</span> &lt;-- adapter (CC2531 in this case) on /dev/ttyACM0</span></pre></div>
<p>As an alternative, the device can also be mapped by an ID. This can be handy if you have multiple serial devices connected to your Raspberry Pi. In the example below the device location is: <code>/dev/serial/by-id/usb-Texas_Instruments_TI_CC2531_USB_CDC___0X00124B0018ED3DDF-if00</code></p>
<div class="highlight highlight-source-shell"><pre>pi@raspberry:/ $ ls -l /dev/serial/by-id
total 0
lrwxrwxrwx. 1 root root 13 Oct 19 19:26 usb-Texas_Instruments_TI_CC2531_USB_CDC___0X00124B0018ED3DDF-if00 -<span class="pl-k">&gt;</span> ../../ttyACM0</pre></div>
<p><strong>NOTE:</strong> Docker mount command does not support certain symbols like colon (Example: <code>/dev/serial/by-id/usb-Silicon_Labs_http:__slae.sh_cc2652_-_slaesh_s_iot_stuff_XX:XX:XX:XX:XX:XX:XX:XX-if00-port0</code>). Error Message: <code>bad format for path</code>. You can create a persistant device name with udev or create a symbolic link.</p>
<h2><a id="user-content-2-installing" class="anchor" aria-hidden="true" href="#2-installing"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>2. Installing</h2>
<div class="highlight highlight-source-shell"><pre><span class="pl-c"><span class="pl-c">#</span> Setup Node.js repository</span>
sudo curl -sL https://deb.nodesource.com/setup_14.x <span class="pl-k">|</span> sudo -E bash -

<span class="pl-c"><span class="pl-c">#</span> NOTE 1: If you see the message below please follow: https://gist.github.com/Koenkk/11fe6d4845f5275a2a8791d04ea223cb.</span>
<span class="pl-c"><span class="pl-c">#</span> ## You appear to be running on ARMv6 hardware. Unfortunately this is not currently supported by the NodeSource Linux distributions. Please use the 'linux-armv6l' binary tarballs available directly from nodejs.org for Node.js 4 and later.</span>
<span class="pl-c"><span class="pl-c">#</span> IMPORTANT: In this case instead of the apt-get install mentioned below; do: sudo apt-get install -y git make g++ gcc</span>

<span class="pl-c"><span class="pl-c">#</span> NOTE 2: On x86, Node.js 10 may not work. It's recommended to install an unofficial Node.js 14 build which can be found here: https://unofficial-builds.nodejs.org/download/release/ (e.g. v14.16.0)</span>

<span class="pl-c"><span class="pl-c">#</span> Install Node.js;</span>
sudo apt-get install -y nodejs git make g++ gcc

<span class="pl-c"><span class="pl-c">#</span> Verify that the correct nodejs and npm (automatically installed with nodejs)</span>
<span class="pl-c"><span class="pl-c">#</span> version has been installed</span>
node --version  <span class="pl-c"><span class="pl-c">#</span> Should output v10.X, v12.X, v14.X or v15.X</span>
npm --version  <span class="pl-c"><span class="pl-c">#</span> Should output 6.X or 7.X</span>

<span class="pl-c"><span class="pl-c">#</span> Clone Zigbee2MQTT repository</span>
sudo git clone https://github.com/Koenkk/zigbee2mqtt.git /opt/zigbee2mqtt
sudo chown -R pi:pi /opt/zigbee2mqtt

<span class="pl-c"><span class="pl-c">#</span> Install dependencies (as user "pi")</span>
<span class="pl-c1">cd</span> /opt/zigbee2mqtt
npm ci --production</pre></div>
<p>If everything went correctly the output of <code>npm ci</code> is similar to (the number of packages and seconds is probably different on your device):</p>
<div class="highlight highlight-source-shell"><pre>node-pre-gyp info ok
added 383 packages <span class="pl-k">in</span> 111.613s</pre></div>
<p>Note that the <code>npm ci</code> produces some <code>warning</code> which can be ignored.</p>
<h2><a id="user-content-3-configuring" class="anchor" aria-hidden="true" href="#3-configuring"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>3. Configuring</h2>
<p>Before we can start Zigbee2MQTT we need to edit the <code>configuration.yaml</code> file. This file contains the configuration which will be used by Zigbee2MQTT.</p>
<p>Open the configuration file:</p>
<div class="highlight highlight-source-shell"><pre>nano /opt/zigbee2mqtt/data/configuration.yaml</pre></div>
<p>For a basic configuration, the default settings are probably good. The only thing we need to change is the MQTT server url and authentication (if applicable). This can be done by changing the section below in your <code>configuration.yaml</code>.</p>
<div class="highlight highlight-source-yaml"><pre><span class="pl-c"><span class="pl-c">#</span> MQTT settings</span>
<span class="pl-ent">mqtt</span>:
  <span class="pl-c"><span class="pl-c">#</span> MQTT base topic for Zigbee2MQTT MQTT messages</span>
  <span class="pl-ent">base_topic</span>: <span class="pl-s">zigbee2mqtt</span>
  <span class="pl-c"><span class="pl-c">#</span> MQTT server URL</span>
  <span class="pl-ent">server</span>: <span class="pl-s"><span class="pl-pds">'</span>mqtt://localhost<span class="pl-pds">'</span></span>
  <span class="pl-c"><span class="pl-c">#</span> MQTT server authentication, uncomment if required:</span>
  <span class="pl-c"><span class="pl-c">#</span> user: my_user</span>
  <span class="pl-c"><span class="pl-c">#</span> password: my_password</span></pre></div>
<p>Save the file and exit.</p>
<p>It is recommended to use a custom network key. This can be done by adding the following to your <code>configuration.yaml</code>. With this Zigbee2MQTT will generate a network key on next startup.</p>
<div class="highlight highlight-source-yaml"><pre><span class="pl-ent">advanced</span>:
    <span class="pl-ent">network_key</span>: <span class="pl-s">GENERATE</span></pre></div>
<p>This can be done with the following command:</p>

(Optional) Running as a daemon with systemctl</h2>
<p>To run Zigbee2MQTT as daemon (in background) and start it automatically on boot we will run Zigbee2MQTT with systemctl.</p>
<div class="highlight highlight-source-shell"><pre><span class="pl-c"><span class="pl-c">#</span> Create a systemctl configuration file for Zigbee2MQTT</span>
sudo nano /etc/systemd/system/zigbee2mqtt.service</pre></div>
<p>Add the following to this file:</p>
<pre><code>[Unit]
Description=zigbee2mqtt
After=network.target

[Service]
ExecStart=/usr/bin/npm start
WorkingDirectory=/opt/zigbee2mqtt
StandardOutput=inherit
#or use StandardOutput=null if you don't want Zigbee2MQTT messages filling syslog, for more options see systemd.exec(5)
StandardError=inherit
Restart=always
User=pi

[Install]
WantedBy=multi-user.target
</code></pre>
<p><strong>NOTE:</strong> If you are using a Raspberry Pi 1 or Zero AND if you followed this <a href="https://gist.github.com/Koenkk/11fe6d4845f5275a2a8791d04ea223cb">guide</a>, replace <code>ExecStart=/usr/bin/npm start</code> with <code>ExecStart=/usr/local/bin/npm start</code>.</p>
<p>Save the file and exit.</p>
<p>Verify that the configuration works:</p>
<div class="highlight highlight-source-shell"><pre><span class="pl-c"><span class="pl-c">#</span> Start Zigbee2MQTT</span>
sudo systemctl start zigbee2mqtt

<span class="pl-c"><span class="pl-c">#</span> Show status</span>
systemctl status zigbee2mqtt.service</pre></div>
<p>Output should look like:</p>

(For later) Update Zigbee2MQTT to the latest version</h2>
<p>To update Zigbee2MQTT to the latest version, execute:</p>
<div class="highlight highlight-source-shell"><pre><span class="pl-c"><span class="pl-c">#</span> Stop Zigbee2MQTT and go to directory</span>
sudo systemctl stop zigbee2mqtt

<span class="pl-c"><span class="pl-c">#</span> Backup configuration</span>

<span class="pl-c"><span class="pl-c">#</span> Update</span>
git checkout HEAD -- npm-shrinkwrap.json
git pull
npm ci

<span class="pl-c"><span class="pl-c">#</span> Restore configuration</span>
cp -R data-backup/<span class="pl-k">*</span> data
